# 开发环境数据目录挂载

`make dev-start` 与 `./scripts/dev.sh start` 支持两种开发数据模式：

- 未设置 `WEKNORA_DEV_DATA_ROOT`：沿用历史 Docker named volumes。
- 显式设置 `WEKNORA_DEV_DATA_ROOT`：把持久化数据绑定到宿主机目录，推荐默认值为 `/data/weknora`。

## 默认规则

- 默认行为：不设置 `WEKNORA_DEV_DATA_ROOT`，继续使用 Docker named volumes
- 推荐宿主机目录：`/data/weknora`
- 自定义目录：`make dev-start WEKNORA_DEV_DATA_ROOT=/data/weknora/custom-dev`
- 脚本直调：`WEKNORA_DEV_DATA_ROOT=/data/weknora ./scripts/dev.sh start`
- `make dev-stop` 不会删除宿主机数据，也不会删除 named volume

## 主要目录映射

| 服务 | 宿主机路径 |
|------|------------|
| PostgreSQL | `${WEKNORA_DEV_DATA_ROOT}/postgres` |
| Redis | `${WEKNORA_DEV_DATA_ROOT}/redis` |
| MinIO | `${WEKNORA_DEV_DATA_ROOT}/minio` |
| Qdrant | `${WEKNORA_DEV_DATA_ROOT}/qdrant` |
| OpenSearch | `${WEKNORA_DEV_DATA_ROOT}/opensearch` |
| Milvus | `${WEKNORA_DEV_DATA_ROOT}/milvus` |
| Neo4j | `${WEKNORA_DEV_DATA_ROOT}/neo4j` |
| DocReader | `${WEKNORA_DEV_DATA_ROOT}/docreader/tmp` |
| Jaeger | `${WEKNORA_DEV_DATA_ROOT}/jaeger` |
| SearXNG 配置 | `${WEKNORA_DEV_DATA_ROOT}/searxng/config` |
| Langfuse ClickHouse 数据 | `${WEKNORA_DEV_DATA_ROOT}/langfuse/clickhouse/data` |
| Langfuse ClickHouse 日志 | `${WEKNORA_DEV_DATA_ROOT}/langfuse/clickhouse/logs` |
| Langfuse MinIO | `${WEKNORA_DEV_DATA_ROOT}/langfuse/minio` |

## 权限说明

启动脚本会自动创建所需目录，并对这些子目录执行 `chmod 0777`，以减少不同镜像用户导致的写入失败。

如果 `/data/weknora` 不可写，请先处理权限，或者改用可写的自定义目录。

## 历史数据说明

旧版 Docker named volume 不会被自动迁移或删除。未设置 `WEKNORA_DEV_DATA_ROOT` 时，当前脚本仍会继续使用这些 volume；需要切到宿主机目录模式时，请先自行迁移旧数据。
