# WeKnora 项目功能清单

## 1. 说明

本文档基于当前仓库的入口文档、后端路由、前端路由、CLI 命令面、SDK 接口目录、DocReader 说明以及小程序与部署目录进行整理，目标是给出一份可供产品、研发、测试和实施团队共用的功能清单。

本次梳理重点参考了以下实现锚点：

- 根目录 README 与 docs 下的产品/开发文档
- 后端入口 `cmd/server/main.go` 与路由注册 `internal/router/router.go`
- 后端功能处理器 `internal/handler`、应用服务 `internal/application/service`
- 前端路由 `frontend/src/router/index.ts` 与 `views` / `api` / `stores` 目录
- CLI 文档 `cli/README.md` 与 `cli/cmd` 命令结构
- Go 客户端 SDK `client/`
- 文档解析服务 `docreader/`
- MCP 服务器 `mcp-server/`
- 微信小程序 `miniprogram/`

说明边界如下：

- 这里描述的是“项目已经提供或明显预留的功能能力”，不是逐文件 API 逐项抄录。
- 一级模块按业务能力和交付形态划分；二级模块按用户可感知功能或明确服务职责划分。
- 某些功能跨越前后端、异步任务和外部依赖，文中会将它们归到主要责任模块下说明。

## 2. 项目整体定位

WeKnora 是一套围绕企业知识治理、RAG 问答、Agent 推理和 Wiki 化沉淀构建的知识平台。系统的核心不是单一聊天能力，而是把“文档导入、解析、切分、索引、检索、问答、工具调用、知识沉淀、权限控制、渠道接入、运维部署”串成一条完整链路。

从交付形态上看，仓库同时提供 Web 控制台、后端 API、CLI、Go SDK、DocReader 文档解析服务、MCP Server、小程序与部署资产；从业务能力上看，系统重点覆盖知识库管理、会话问答、Agent 编排、数据源同步、模型与检索配置、多租户权限、公开分享、Wiki/知识图谱和外部 IM 接入等场景。

## 4. 建议作为对外介绍重点突出的能力组合

如果后续需要将本文档进一步整理成对外介绍、实施清单或测试用例目录，建议优先围绕以下组合来组织：

1. 知识导入链路：上传/同步 -> DocReader 解析 -> 分块调试 -> 索引构建 -> RAG 问答。
2. Agent 能力链路：自定义 Agent -> 技能/工具装配 -> MCP 工具接入 -> 推理与审批 -> 页面分享。
3. 企业治理链路：认证/OIDC -> 多租户 -> 组织树 -> RBAC -> 审计日志 -> 系统设置。
4. 多端交付链路：Web 控制台 -> CLI -> Go SDK -> MCP Server -> 小程序 -> IM 渠道。
5. 知识沉淀链路：会话导出 -> 长文档生成 -> Wiki 页面 -> 知识图谱 -> 共享传播。
