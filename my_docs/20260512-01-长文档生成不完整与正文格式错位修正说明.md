# 长文档生成不完整与正文格式错位修正说明

## 问题依据

参考 `my_docs/test/15.md` 中 2026-05-12 16:07:58 发起的 `full_document` 生成日志，本次请求进入 `knowledge_grounded_full_document` 路径。最终日志显示：

- `completion_status: partial`
- `finish_reason: markdown_quality_failed`
- `document_generation_status: needs_review`
- `outline_sections: 7`
- `runtime_sections: 1`
- `artifact_registered: false`

这说明生成链路不是只在前端显示时被截断，而是在后端第一章完成后被 Markdown 质量门判定失败，随后中断了剩余 6 个章节的生成，因此用户看到的结果不完整。

## 根因分析

### 1. Markdown 质量门与生成提示不一致

章节生成提示允许模型使用 H3/H4 标题展开小节，但质量校验器将 H4 及更深层级统一判为非法。这会导致正常的 H4 小标题被误判为 `markdown_quality_failed`，进而把完整文档状态降级为 `partial / needs_review`。

### 2. 质量门失败被当作硬中断

长文档章节生成完成后会调用 `applyGeneratedSectionMarkdownQualityGate`。旧逻辑在 `qualityOK=false` 时立即设置：

- `completionStatus = partial`
- `finishReason = markdown_quality_failed`
- `documentGenerationStatus = needs_review`

随后跳出章节循环，导致仅完成第 1 章就停止。

### 3. 前端优先展示流式原始片段

后端在章节流式输出过程中先把模型原始 token 发给前端，章节结束后才执行 Markdown 归一化和修复。旧前端在收到完成事件后仍优先展示已累积的流式原始片段，没有使用完成事件中的规范化 `final_answer` 覆盖，因此会继续显示标题粘连、列表缩进错位等问题。

### 4. 正文已开始后状态仍可能显示规划中

前端阶段标签主要从最新 thinking 事件推断。长文档场景下，大纲卡片中的 planning 事件可能仍作为可见阶段来源，导致正文已经开始输出时仍显示“规划中”。

## 已落地修复

### 后端修复

1. 放宽 Markdown 质量校验规则：允许 H4 作为章节内细分标题，仅拒绝 H5 及更深标题；H3 仍按规划小节编号校验。
2. 长文档章节质量门不再作为整篇生成的硬中断条件；当修复失败或仍存在质量问题时，继续生成后续章节，并将问题写入 `quality_issues` 供审查。
3. 质量告警章节也会补齐章节间空行，避免下一章标题与上一章正文粘连。
4. `AgentStreamHandler` 在长文档完成事件中优先采用后端规范化后的 `FinalAnswer`，保证数据库持久化和完成事件 payload 都使用修正后的权威正文。

### 前端修复

1. `AgentStreamDisplay.vue` 在会话完成且 `agent_complete.final_answer` 存在时，使用完成事件中的权威正文替换流式原始片段，修复最终正文格式错位。
2. 当正文已经开始流式输出但会话尚未完成时，加载状态优先显示“生成中”，避免继续显示“规划中”。

## 验证结果

已通过后端聚焦回归测试：

```bash
go test -v ./internal/application/service ./internal/handler/session -run 'TestValidateGeneratedSectionMarkdownAllowsH4Details|TestRunDedicatedFullDocumentGenerationPath_ContinuesWhenMarkdownQualityGateWarns|TestHandleComplete_PrefersAuthoritativeLongDocumentAnswer|TestRunKnowledgeGroundedFullDocumentGenerationPath_PersistsGenerationRun|TestRunDedicatedFullDocumentGenerationPath_NormalizesFinalAnswerAndEmitsQualityIssues'
```

结果：全部通过。

前端执行 `pnpm type-check` 时失败，但失败点来自仓库既有的其他组件类型问题，包括 `IMChannelPanel.vue`、`manual-knowledge-editor.vue`、`MarkdownTestPage.vue`、`ApiInfo.vue`、`KnowledgeBase.vue`、`DocumentListView.vue` 等，与本次修改的 `AgentStreamDisplay.vue` 无直接关系；当前修改文件经编辑器诊断未发现错误。

## 修复后预期效果

1. 首轮完整文档不再因为普通 Markdown 质量告警在第 1 章后停止。
2. H4 小标题不会被误判为非法格式。
3. 最终展示内容以归一化后的权威全文为准，标题、列表、加粗标签和章节间距更稳定。
4. 正文开始输出后，状态标签应显示“生成中”或后续终态，不再停留在“规划中”。
