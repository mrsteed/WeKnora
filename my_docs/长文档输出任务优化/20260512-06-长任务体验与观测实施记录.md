# 阶段五：长任务体验与观测实施记录

## 1. 本次落地范围

本次阶段五落地聚焦两个目标：

1. 让长文档任务在页面上持续可见，用户能明确知道当前所处阶段，以及当前正在处理哪一章。
2. 让长文档任务在后端日志里形成可追踪的完成摘要与索引结果，便于运维和验收排查。

## 2. 已落地改动

### 2.1 前端体验增强

修改文件：[frontend/src/views/chat/components/AgentStreamDisplay.vue](frontend/src/views/chat/components/AgentStreamDisplay.vue)

已补充：

1. 规划卡片新增“当前章节进度”展示。
2. 流式加载区新增章节级进度展示。
3. 章节进度支持识别以下场景：
   - 正在检索第 N/M 章
   - 正在生成第 N/M 章
   - 第 N/M 章已完成，正在收尾
   - 检索过程中的子查询进度（如 1/4）

页面效果上，长任务现在会同时展示：

1. 阶段状态：规划、检索、生成、收尾。
2. 当前章节：第几章 / 共几章。
3. 当前章节标题。
4. 检索阶段的子查询进度。

### 2.2 后端观测增强

修改文件：[internal/application/service/session_agent_qa.go](internal/application/service/session_agent_qa.go)

已补充长文档完成摘要日志，统一记录：

1. completion_status
2. finish_reason
3. failure_reason
4. document_generation_status
5. total_duration_ms
6. total_steps
7. outline_title
8. outline_sections
9. runtime_sections
10. runtime_budget_adjusted

12. auto_continue_next

该日志用于回答“这一轮长任务最终有没有完成、总耗时多少、规划了多少章节、运行时是否触发预算收缩”。

### 2.3 历史索引结果观测增强

修改文件：[internal/application/service/message.go](internal/application/service/message.go)

已增强聊天历史索引日志，记录：

1. task_kind
2. completion_status
3. document_generation_status
4. artifact_id
5. passage_count
6. knowledge_id

该日志用于回答“长文档最终是否进入历史索引、采用了哪种索引任务类型、关联了哪个 artifact、最终生成了哪个 knowledge 记录”。

### 2.4 审查后补充修正

本轮复审发现两个观测缺口，并已修正：

1. 原实现虽然分别记录了 artifact 注册日志和索引成功日志，但缺少一条统一的长文档收尾摘要，不能直接从单条日志看出“本轮完成后是否注册 artifact、是否计划进入索引”。现已在 assistant completion 收尾处补充 `[LongDocument] persistence summary`。
2. 原实现对“未进入聊天历史索引”的场景缺少明确 skipped 原因。现已在索引服务中补充 skipped 日志，区分：
   - `allow_indexing_disabled`
   - `completion_status_partial` 等完成态限制
   - `planning_content`
   - `chat_history_kb_not_configured`
   - `no_passages`

这样阶段五的观测链路现在可以覆盖：完成事件、artifact 注册、索引计划、索引跳过原因、索引成功结果。

## 3. 手工验证记录项

后续进行端到端联调时，建议按下表记录：

| 维度 | 记录项 | 说明 |
| --- | --- | --- |
| 基本信息 | session_id / message_id | 用于串联 SSE、artifact、索引日志 |
| 规划结果 | outline_title / outline_sections | 确认规划是否生成成功 |
| 运行耗时 | total_duration_ms | 以 `agent_complete` 与后端完成摘要日志为准 |
| 章节进度 | 第 N/M 章、章节标题 | 确认前端展示与后端进度一致 |
| 完成状态 | completion_status / document_generation_status | 判断 completed / partial / blocked |
| 产物注册 | artifact_id / revision_no | 确认最终文档已注册 artifact |
| 索引结果 | task_kind / passage_count / knowledge_id | 确认是否进入聊天历史索引 |
