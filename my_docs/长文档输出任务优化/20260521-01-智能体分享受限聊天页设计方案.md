# 智能体分享受限聊天页设计方案

## 1. 背景与目标

当前系统已经具备智能体配置、组织内共享智能体、标准聊天页、多租户权限隔离等能力，但还缺少一种“把某个智能体以独立聊天页形式分享出去”的能力。

本次需求可以拆成三个明确目标：

1. 新增一种面向聊天入口的智能体分享能力，而不只是当前的“共享到组织”。
2. 分享后访问者进入一个独立聊天页面，只能围绕该智能体既有设计进行问答。
3. 在该页面内不允许做任何设置类操作，包括但不限于切换智能体、切换模型、切换知识库、开启或关闭网络搜索、打开系统设置。
4. 分享链接必须支持匿名访问，外部访问者打开链接后不需要登录、不需要 API Key，也不需要加入任何组织。

本方案只输出设计，不直接修改业务代码。

## 2. 现状梳理

### 2.1 智能体分享现状

当前仓库中已经存在的“智能体分享”能力，本质上是组织协作能力，而不是页面分享能力。

现有证据如下：

1. `docs/api/agent.md` 明确说明智能体共享与跨组织分发属于组织管理域，不在智能体 CRUD 文档内。
2. `docs/api/organization.md` 已定义 `POST /api/v1/agents/:id/shares`、`GET /api/v1/agents/:id/shares`、`DELETE /api/v1/agents/:id/shares/:share_id`、`GET /api/v1/organizations/:id/agent-shares`。
3. `frontend/src/components/AgentShareSettings.vue` 当前 UI 也是把智能体共享给某个组织，并未生成聊天链接，也未生成独立聊天页。
4. `frontend/src/stores/organization.ts` 的 `fetchSharedAgents()` 拉取的是“共享给我”的组织内智能体列表。

结论：当前系统已有“资源共享”，没有“分享聊天页”。

### 2.2 聊天页面现状

当前正式聊天入口是标准平台页的一部分。

现有证据如下：

1. `frontend/src/router/index.ts` 当前聊天路由为 `/platform/chat/:chatid`，要求初始化完成且已登录。
2. `frontend/src/views/platform/index.vue` 会统一挂载左侧菜单、全局设置弹窗、全局命令面板。
3. `frontend/src/views/chat/index.vue` 是标准聊天主视图。
4. `frontend/src/components/Input-field.vue` 负责输入区、智能体切换、知识库选择、模型切换、网络搜索开关、上传入口等。

结论：当前聊天页天然属于平台壳内页面，默认暴露平台级导航与设置能力，不适合作为“受限分享聊天页”直接复用。

### 2.3 现有嵌入式能力与不足

当前聊天视图已经有一定的受控上下文能力，但还不足以满足本需求。

现有证据如下：

1. `frontend/src/views/chat/index.vue` 已支持 `agentId`、`kbIds`、`embeddedMode` 等 props。
2. `frontend/src/components/Input-field.vue` 在模板中通过 `v-if="!embeddedMode"` 隐藏了左侧部分控制区。
3. 但同一组件内部仍大量依赖 `settingsStore` 的全局状态，例如 `selectedAgentId`、`selectedKnowledgeBases`、`selectedFiles`、`selectedAgentSourceTenantId`、模型选择、网络搜索等。
4. 组件内部仍保留了跳转 `/platform/settings`、打开设置分区、拉取全部智能体列表、切换共享智能体等逻辑。

结论：当前 `embeddedMode` 只是一个 UI 层面的部分隐藏开关，不是完整的“受限运行时模式”。如果直接复用，访问者仍可能通过全局 store 或现有跳转路径突破页面边界。

## 3. 需求边界

### 3.1 本次覆盖范围

本方案覆盖以下内容：

1. 为智能体新增“页面分享”能力。
2. 分享后生成一个独立访问地址。
3. 访问地址打开后进入一个只绑定当前智能体的聊天页面。
4. 页面内严格按智能体既有配置进行问答。
5. 页面内不提供任何设置能力。
6. 页面内支持多轮对话与推荐问题，但是否启用历史记忆、知识库、图片上传、附件上传、网络搜索等，都完全由智能体配置决定。
7. 分享链接匿名可访问，访问侧接口不依赖登录态、租户切换、组织成员身份或 API Key。

### 3.2 本次不覆盖范围

以下内容不纳入 P0：

1. 替换现有组织共享能力。
2. 分享页上的智能体在线编辑。
3. 访问者在分享页自行切换模型、知识库、MCP、Web Search。
4. 把分享页做成完整平台页的镜像副本。
5. 让匿名访问者浏览租户、组织、知识库、智能体、历史会话等平台资源列表。

说明：本需求已经明确要求“打开分享链接不需要任何认证”，因此 P0 必须包含匿名访客会话、公开访问接口和风控能力。管理侧仍然需要登录和权限校验，只有访问侧分享链接免认证。

## 4. 设计原则

### 4.1 运行时隔离原则

分享聊天页必须使用独立运行时上下文，不能直接依赖标准聊天页的全局 `settingsStore` 作为唯一事实来源。

### 4.2 智能体主导原则

分享聊天页的所有能力开关都以智能体当前配置为准，包括：

1. 运行模式。
2. 系统提示词。
3. 模型选择。
4. 知识库范围。
5. MCP 服务范围。
6. Web Search 是否开启。
7. 图片与附件能力。
8. 多轮对话能力。

访问者只能提问，不能重新配置这些能力。

### 4.3 前后端双重约束原则

“无法做设置工作”不能只靠前端隐藏按钮实现，后端也必须忽略或拒绝来自分享页的越界设置字段。

### 4.4 与现有能力兼容原则

尽量复用当前聊天渲染、消息流式输出、推荐问题、智能体执行链路，避免新写一套平行聊天系统。

## 5. 总体方案概览

新增“智能体页面分享”能力，与当前“共享到组织”并列存在。

整体链路如下：

1. 创建者在智能体管理页开启“页面分享”。
2. 后端为该智能体生成一个分享记录和一个稳定分享码。
3. 前端展示复制链接、打开页面、关闭分享等操作。
4. 访问者打开分享链接后，进入独立的分享聊天页。
5. 分享聊天页先通过分享码拉取当前智能体摘要和受限运行时配置。
6. 分享聊天页创建一个匿名访客会话，后端返回 `session_id` 和只用于当前分享会话的 `visitor_token`。
7. 页面使用“受限聊天上下文”驱动聊天组件，只允许发送问题。
8. 后端在每次对话请求中按分享记录反解出目标智能体，并以该智能体当前配置执行问答。
9. 所有设置类字段由后端固定，不采纳访问者传入值。

本方案建议采用“分享记录绑定智能体，运行时跟随智能体最新配置”的模式，而不是在分享时固化一份完整配置快照。

原因如下：

1. 与当前组织共享语义保持一致，分享后的范围变化可以自动同步。
2. 满足“只能根据当前智能体的设计进行问答”的要求。
3. 避免每次智能体修改都要重新生成分享版本。

## 6. 功能设计

### 6.1 分享管理能力

在智能体管理页新增“页面分享”入口，和现有组织共享区分开。

建议前端交互如下：

1. 智能体详情或编辑弹窗中新增“页面分享”分区。
2. 未开启时，显示“开启分享”按钮。
3. 开启后，展示分享状态、分享链接、复制按钮、在新窗口打开按钮、关闭分享按钮。
4. 可选展示最近访问时间与创建时间。

P0 建议一条智能体仅维护一条有效页面分享记录，避免一个智能体同时生成多条链接带来的管理复杂度。

### 6.2 分享聊天页能力

分享聊天页只提供以下最小能力：

1. 展示智能体头像、名称、描述。
2. 展示欢迎语或推荐问题。
3. 展示消息列表。
4. 提供输入框发送问题。
5. 如果智能体允许，则展示图片上传或附件上传。
6. 如果智能体支持多轮，则在当前分享会话内保留上下文。

分享聊天页不提供以下能力：

1. 智能体切换。
2. 模型切换。
3. 知识库手动切换或 @ 选择。
4. Web Search 手动开关。
5. 打开全局设置。
6. 打开系统命令面板。
7. 跳转到智能体管理页进行编辑。
8. 查看其他聊天会话列表。

### 6.3 聊天行为规则

分享聊天页的问答规则如下：

1. `agent_id` 固定为当前分享绑定的智能体。
2. `source_tenant_id` 固定为分享智能体所属租户。
3. 知识库范围严格按该智能体配置推导，访问者不能额外增删。
4. Web Search 是否可用由智能体配置决定，访问者不能手动开关。
5. 模型由智能体配置或系统默认规则决定，访问者不能切换。
6. 若智能体禁用图片上传、附件上传、音频上传，对应入口不显示，后端也拒绝相关输入。
7. 若智能体禁用多轮对话，则每次请求都按单轮模式执行，前端不展示“延续历史”的能力。

## 7. 数据模型设计

### 7.1 新增 `agent_page_shares` 表

建议新增页面分享表，专门表达“一个智能体对应一个受限聊天入口”。

建议字段如下：

```sql
id                  varchar(36)    primary key
agent_id            varchar(36)    not null
source_tenant_id    bigint         not null
share_code          varchar(64)    not null unique
access_scope        varchar(16)    not null default 'anonymous'
status              varchar(16)    not null default 'active'
created_by          varchar(36)    not null
anonymous_session_limit integer     not null default 0
rate_limit_per_minute integer       not null default 0
last_accessed_at    timestamp      null
expires_at          timestamp      null
created_at          timestamp      not null
updated_at          timestamp      not null
deleted_at          timestamp      null
```

字段说明：

1. `agent_id`：被分享的智能体 ID。
2. `source_tenant_id`：智能体所属租户，用于执行阶段解析模型、知识库与 MCP。
3. `share_code`：对外暴露的稳定分享码，不直接使用数据库主键。
4. `access_scope`：P0 固定为 `anonymous`，表示打开分享链接无需登录或 API Key。
5. `status`：建议支持 `active`、`disabled`、`expired`。
6. `anonymous_session_limit`：可选的匿名会话总量限制，`0` 表示不限制。
7. `rate_limit_per_minute`：可选的分享级限流配置，`0` 表示使用系统默认值。
8. `expires_at`：P0 可先保留字段，不强制前端提供配置。

索引建议：

1. `uniq_agent_page_shares_share_code`：唯一索引。
2. `idx_agent_page_shares_agent_id`：按智能体查询。
3. `idx_agent_page_shares_source_tenant_id`：按租户查询。

### 7.2 会话侧建议

分享聊天页不建议新建一套平行消息表，优先复用现有会话与消息体系。但由于访问者没有登录态，必须补充匿名会话标识，不能依赖 `user_id` 做权限判断。

建议在现有会话模型上补充可选字段：

1. `access_mode`：`platform` 或 `agent_share_page`。
2. `agent_page_share_id`：指向页面分享记录。
3. `anonymous_visitor_id`：匿名访客 ID，可由后端生成并写入浏览器本地存储。
4. `visitor_token_hash`：匿名会话访问令牌哈希，只保存哈希，不保存明文令牌。
5. `visitor_ip_hash`：访问 IP 哈希，用于风控和审计，不直接存明文 IP。
6. `user_agent_hash`：浏览器 UA 哈希，用于辅助识别异常访问。

这样做的目的：

1. 继续复用现有消息流、历史消息查询、标题生成与流式输出机制。
2. 在权限校验时识别“该会话来自哪个分享页”。
3. 避免标准平台聊天页误打开分享页会话。
4. 在无登录态条件下，仍能保证匿名访问者只能读写自己通过该分享链接创建的会话。

### 7.3 匿名会话令牌设计

分享页免认证不等于会话无边界。建议在创建匿名会话时生成一个高强度随机 `visitor_token`，前端仅保存在当前浏览器本地，后续读取消息、继续输出和发送问题时通过请求头携带。

建议规则如下：

1. `visitor_token` 只在创建会话时返回一次，后端只保存哈希值。
2. 后续请求使用 `X-Share-Session-Token` 携带令牌。
3. `session_id` 与 `visitor_token` 必须同时匹配，才能访问该匿名会话。
4. 分享关闭、过期或会话过期后，令牌立即失效。
5. 不把该令牌视为用户身份认证，只把它作为匿名会话读写凭证。

## 8. 接口设计

### 8.1 管理侧接口

建议新增以下接口：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/agents/:id/page-share` | 获取当前智能体的页面分享状态 |
| `POST` | `/api/v1/agents/:id/page-share` | 创建或重新启用页面分享 |
| `DELETE` | `/api/v1/agents/:id/page-share` | 关闭页面分享 |

管理侧接口必须继续走现有登录认证与智能体管理权限校验。只有智能体拥有者或具备管理权限的用户可以创建、查看和关闭分享。

`POST /api/v1/agents/:id/page-share` 响应建议：

```json
{
  "success": true,
  "data": {
    "id": "aps-00000001",
    "agent_id": "agent-00000001",
    "source_tenant_id": 1,
    "share_code": "8YwQ3N4mK2Lx",
    "status": "active",
    "access_scope": "anonymous",
    "share_url": "/share/agents/8YwQ3N4mK2Lx",
    "created_at": "2026-05-21T10:00:00+08:00"
  }
}
```

### 8.2 访问侧接口

建议新增以下接口：

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| `GET` | `/api/v1/public/agent-page-shares/:share_code` | 免认证获取分享页元信息与受限运行时摘要 |
| `POST` | `/api/v1/public/agent-page-shares/:share_code/sessions` | 免认证创建匿名分享页会话 |
| `GET` | `/api/v1/public/agent-page-shares/:share_code/sessions/:session_id/messages` | 通过匿名会话令牌获取该分享会话消息 |
| `POST` | `/api/v1/public/agent-page-shares/:share_code/chat` | 通过匿名会话令牌发起分享页问答 |
| `POST` | `/api/v1/public/agent-page-shares/:share_code/chat/continue` | 通过匿名会话令牌继续流式输出 |

访问侧接口必须挂在明确的 public 路由组下，不经过登录认证中间件，也不要求 API Key。该路由组只能暴露分享页所需的最小能力，不能复用平台内需要登录态的通用资源接口。

其中 `GET /api/v1/public/agent-page-shares/:share_code` 的核心作用不是返回完整 agent 配置，而是返回“分享页渲染所需的最小只读摘要”。

建议响应结构：

```json
{
  "success": true,
  "data": {
    "share": {
      "id": "aps-00000001",
      "share_code": "8YwQ3N4mK2Lx",
      "status": "active",
      "access_scope": "anonymous"
    },
    "agent": {
      "id": "agent-00000001",
      "name": "智能客服助手",
      "description": "基于售后知识库提供问答",
      "avatar": "🤖"
    },
    "runtime": {
      "agent_mode": "smart-reasoning",
      "multi_turn_enabled": true,
      "image_upload_enabled": false,
      "audio_upload_enabled": false,
      "attachment_upload_enabled": true,
      "show_web_search_toggle": false,
      "show_model_selector": false,
      "show_kb_selector": false,
      "show_agent_selector": false
    },
    "suggested_questions": [
      "如何申请售后服务？",
      "保修政策是什么？"
    ]
  }
}
```

`POST /api/v1/public/agent-page-shares/:share_code/sessions` 建议响应：

```json
{
  "success": true,
  "data": {
    "session_id": "session-00000001",
    "anonymous_visitor_id": "visitor-00000001",
    "visitor_token": "hS8n...random...Qp2",
    "expires_at": "2026-05-22T10:00:00+08:00"
  }
}
```

`visitor_token` 是匿名会话访问凭证，只返回一次。前端后续请求通过 `X-Share-Session-Token` 传递。

### 8.3 问答接口约束

`POST /api/v1/public/agent-page-shares/:share_code/chat` 请求体建议只保留最小必要字段：

```json
{
  "session_id": "session-00000001",
  "query": "请介绍售后流程",
  "images": [],
  "attachment_uploads": []
}
```

请求头必须包含：

```http
X-Share-Session-Token: hS8n...random...Qp2
```

后端不接受以下来自访问者的自由设置字段：

1. `agent_id`
2. `agent_enabled`
3. `knowledge_base_ids`
4. `knowledge_ids`
5. `enable_memory`
6. `web_search_enabled`
7. `model_id`
8. `rerank_model_id`
9. `selectedAgentSourceTenantId` 等任何前端可构造的变体字段

这些字段应全部由后端根据 `share_code -> agent_page_share -> agent` 反解后自动填充。

### 8.4 匿名访问错误语义

访问侧建议统一错误语义：

1. 分享码不存在、已关闭、已过期：`404 Not Found`，避免暴露分享记录存在性。
2. 缺少或错误的匿名会话令牌：`403 Forbidden`。
3. 匿名会话过期：`410 Gone`。
4. 命中限流：`429 Too Many Requests`。
5. 上传内容不符合智能体配置：`400 Bad Request`。

## 9. 前端设计

### 9.1 路由设计

建议新增平台外路由：

| 路径 | 名称 | 说明 |
| --- | --- | --- |
| `/share/agents/:shareCode` | `agentShareChat` | 页面分享聊天页 |

该路由不应挂在 `/platform` 下。

该路由应配置为 `requiresAuth: false`、`requiresInit: false`，并且不能触发自动跳转登录。页面初始化只依赖分享码公开接口，不读取用户登录态作为必要条件。

原因如下：

1. `/platform` 会自动挂载菜单、设置、命令面板，不符合受限页要求。
2. 分享页应是独立壳层，避免把平台级能力带进来。
3. 页面结构更容易做到“只聊天，不配置”。

### 9.2 页面结构设计

分享页不新做一套聊天页面，而是新增一个“分享页外壳”，在外壳中最大程度复用现有对话页面。

建议新增以下轻量页面组件：

1. `frontend/src/views/share/AgentShareChatPage.vue`
2. `frontend/src/views/share/components/AgentShareHeader.vue`
3. `frontend/src/views/share/components/AgentShareEmptyState.vue`

页面布局建议：

1. 顶部为智能体信息条，展示头像、名称、描述、分享状态提示。
2. 中部直接挂载 `frontend/src/views/chat/index.vue`，复用现有消息列表、推荐问题、流式输出、artifact 展示、重试、停止生成、自动滚动等能力。
3. 底部继续使用 `frontend/src/components/Input-field.vue`，但由分享页传入“锁定运行时上下文”，隐藏设置类控件并切换到 public 分享接口。

`AgentShareChatPage.vue` 的职责应尽量薄，只负责：

1. 读取 `shareCode`。
2. 调用 public 分享元信息接口。
3. 创建或恢复匿名分享会话。
4. 构造 `ChatRuntimeContext`。
5. 把 `session_id`、`share_code`、`visitor_token`、固定智能体信息传给现有聊天页。

它不负责重新实现消息渲染、SSE 解析、Markdown 渲染、artifact 卡片、附件展示、滚动定位等成熟能力。

### 9.3 组件复用策略

本需求要求“分享的对话连接页面最大程度复用现有的对话页面功能”。因此前端方案应以复用为主、隔离为辅。

必须复用的现有能力如下：

1. `frontend/src/views/chat/index.vue` 的消息列表和流式输出逻辑。
2. `frontend/src/components/Input-field.vue` 的输入框、发送、停止生成、图片/附件上传基础交互。
3. `frontend/src/views/chat/components/botmsg.vue`、`usermsg.vue`、`AgentStreamDisplay.vue`、`ChatDocumentArtifactCard.vue` 等现有消息展示组件。
4. 推荐问题展示与点击发送逻辑。
5. SSE 流式事件处理、自动滚动、停止生成、失败重试、历史消息加载。
6. Markdown 渲染、代码块展示、文档 artifact 查看、版本链抽屉等现有聊天体验。

只允许新增或改造以下适配层：

1. 分享页外壳：处理匿名访问、分享码、匿名会话令牌。
2. 聊天运行时上下文：告诉现有聊天页当前是 `agent-share-page` 模式。
3. API 适配层：在分享模式下把聊天请求路由到 public 分享接口。
4. 控件权限层：在分享模式下隐藏或禁用所有设置类入口。

不建议新增以下内容：

1. 第二套聊天消息组件。
2. 第二套输入框组件。
3. 第二套 SSE 解析逻辑。
4. 第二套 Markdown 渲染或 artifact 展示逻辑。
5. 第二套会话滚动和重试状态管理。

但不建议继续用单一 `embeddedMode` 布尔值承接全部受限逻辑。

建议把当前聊天运行时抽象为：

```ts
interface ChatRuntimeContext {
  mode: 'platform' | 'embedded' | 'agent-share-page'
  shareCode?: string
  shareSessionToken?: string
  fixedAgentId?: string
  fixedSourceTenantId?: string
  fixedKnowledgeBaseIds?: string[]
  publicChatApiBase?: string
  allowAgentSwitch: boolean
  allowKnowledgeBaseSelect: boolean
  allowModelSelect: boolean
  allowWebSearchToggle: boolean
  allowSettingsNavigation: boolean
  allowCommandPalette: boolean
  allowSessionListNavigation: boolean
  allowConversationHistoryNavigation: boolean
}
```

然后由分享页把该上下文显式传给聊天页和输入组件。现有 `embeddedMode` 可以短期保留为 UI 样式标识，但不能继续承担权限控制语义。

建议 `frontend/src/views/chat/index.vue` 做最小改造：

1. 增加可选 `runtimeContext` prop。
2. 当 `runtimeContext.mode === 'agent-share-page'` 时，消息读取、发送、停止和 continue stream 走 public 分享 API。
3. 原有平台聊天逻辑保持默认路径，避免影响 `/platform/chat/:chatid`。
4. 推荐问题使用 `runtimeContext.fixedAgentId` 和分享元信息返回的建议问题，不再读取全局选中智能体。
5. 历史消息只读取当前匿名分享会话，不读取平台会话列表。

### 9.4 输入区改造要点

`frontend/src/components/Input-field.vue` 需要新增受限模式能力，核心要求如下：

1. 不再从全局 `settingsStore.selectedAgentId` 推导分享页当前智能体。
2. 不再允许调用 `settingsStore.selectAgent()` 切换智能体。
3. 不再允许打开 `AgentSelector`。
4. 不再允许打开 `KnowledgeBaseSelector`。
5. 不再允许打开模型选择器。
6. 不再允许跳转 `/platform/settings`。
7. 不再允许触发 `uiStore.openSettings(...)`。
8. 上传能力是否显示，完全读取分享页运行时摘要。

简化说，分享页输入区只保留“输入问题”和“按智能体允许的输入附件种类上传文件”两类能力。

为保证复用，`Input-field.vue` 不建议复制出 `ShareInputField.vue`。推荐在现有组件中增加可选 `runtimeContext` prop，并把当前依赖全局 `settingsStore` 的位置收敛到几个计算属性：

1. `effectiveAgentId`：平台模式来自 `settingsStore.selectedAgentId`，分享模式来自 `runtimeContext.fixedAgentId`。
2. `effectiveKnowledgeBaseIds`：平台模式来自用户选择，分享模式来自后端按智能体配置解析后的固定范围。
3. `effectiveWebSearchEnabled`：平台模式来自设置或智能体配置，分享模式只读智能体配置。
4. `effectiveModelId`：平台模式允许用户选择，分享模式只读智能体配置。
5. `canOpenSettings` / `canSwitchAgent` / `canSelectKnowledgeBase`：平台模式沿用当前行为，分享模式恒为 false。

### 9.5 推荐问题与新会话

分享页可以继续显示推荐问题，但推荐问题的来源必须固定为当前分享智能体。

会话行为建议如下：

1. 初次进入页面自动创建一个分享页会话。
2. 页面刷新后，允许根据 URL、本地缓存中的 `session_id` 与 `visitor_token` 恢复最近一个该分享页下的匿名会话。
3. 不展示完整历史会话列表，避免用户横向访问其他会话。
4. 如果本地匿名会话令牌失效，则自动创建新的匿名会话，不跳转登录页。

## 10. 后端设计

### 10.1 分享解析服务

建议新增页面分享服务，例如：

1. `AgentPageShareRepository`
2. `AgentPageShareService`

职责如下：

1. 创建与关闭页面分享。
2. 通过 `share_code` 解析分享记录。
3. 校验分享状态是否可用。
4. 反解目标智能体与来源租户。
5. 输出分享页受限运行时摘要。

### 10.2 分享页问答执行

分享页问答不建议新建一套完全独立的 Agent 执行管线，而应复用现有知识问答与 Agent 问答链路。

处理逻辑建议如下：

1. 先通过 `share_code` 拿到 `agent_id` 与 `source_tenant_id`。
2. 校验匿名会话令牌是否匹配当前分享记录与会话。
3. 加载当前智能体完整配置。
4. 把智能体配置映射成标准问答请求需要的固定参数。
5. 丢弃访问者传入的自由配置参数。
6. 调用现有聊天服务执行。

这样可以最大化复用当前能力：

1. 智能体推荐问题。
2. 知识库检索。
3. 共享智能体来源租户解析。
4. SSE 流式输出。
5. 消息落库。

### 10.3 后端强校验点

分享页问答请求进入后，后端必须做以下校验：

1. 分享记录存在且状态为 `active`。
2. 分享对应智能体存在且未删除。
3. 当前匿名会话令牌合法，且只绑定当前分享记录与当前会话。
4. 当前会话确实属于该分享页。
5. 访问者未越权传入设置字段。
6. 上传内容类型与数量符合智能体配置。
7. 当前请求未超过分享级、IP 级、会话级限流阈值。

即使前端已经隐藏控件，后端也必须把设置类字段视为不可信输入。

## 11. 权限与安全设计

### 11.1 访问权限

P0 必须采用“匿名免认证访问”策略。

规则如下：

1. 打开 `/share/agents/:shareCode` 不要求登录。
2. 访问公开分享元信息接口不要求登录。
3. 创建匿名分享会话不要求登录。
4. 发起问答不要求登录，但必须携带该匿名会话的 `visitor_token`。
5. 匿名访问者不能调用任何平台管理接口。
6. 管理侧创建、关闭、查看分享仍然要求登录与权限校验。

### 11.2 数据访问边界

分享页访问者只能借助被分享智能体访问其允许范围内的数据，不能获得租户的通用浏览能力。

因此后端必须坚持以下约束：

1. 只能访问该智能体允许的知识库范围。
2. 只能调用该智能体允许的 MCP 服务。
3. 只能使用该智能体对应的模型配置。
4. 不能因为知道 `share_code` 就获得租户其他资源列表。
5. 不能因为匿名分享会话存在，就读取同租户其他平台会话或其他分享会话。

### 11.3 风险控制

建议同时补充：

1. 分享页访问日志。
2. 请求频率限制。
3. 分享关闭后的即时失效。
4. 分享智能体被删除后的访问拦截。
5. 匿名会话过期策略。
6. IP 与分享码维度的限流策略。
7. 单个分享链接的并发会话数量限制。
8. 大模型调用成本保护，例如每日调用量、每日 token 量或失败率熔断。

## 12. 验收标准

满足以下条件即可视为本需求完成：

1. 智能体可开启页面分享并获得稳定链接。
2. 未登录浏览器打开链接后直接进入独立聊天页，而不是跳转登录页或平台标准聊天页。
3. 页面内无法切换智能体。
4. 页面内无法切换模型。
5. 页面内无法切换知识库或文件范围。
6. 页面内无法打开系统设置。
7. 页面内问答严格使用分享智能体当前配置。
8. 智能体配置变更后，分享页行为随之同步。
9. 关闭分享后，原链接立即不可用。
10. 分享页会话不能在标准聊天页中越权打开或继续配置。
11. 匿名访问者没有登录态、没有 API Key、不是组织成员时仍可发起问答。
12. 未携带正确 `visitor_token` 时，不能读取或继续其他匿名分享会话。

## 13. 开发计划

### 13.1 第一阶段：后端分享基础能力

目标：先让系统具备“创建分享链接”和“匿名解析分享链接”的最小闭环。

主要任务：

1. 新增 `agent_page_shares` 数据模型、repository、service 和迁移脚本。
2. 新增管理侧接口：获取分享状态、开启分享、关闭分享。
3. 管理侧接口复用现有登录认证和智能体管理权限校验。
4. 新增 public 分享元信息接口：通过 `share_code` 返回智能体摘要和受限运行时摘要。
5. 生成不可枚举的高强度 `share_code`，避免使用自增 ID 或 agent ID 作为公开链接。

交付标准：

1. 已登录管理员可为智能体生成 `/share/agents/:shareCode` 链接。
2. 未登录访问者可通过 public 接口读取该分享页所需的最小展示信息。
3. 分享关闭、智能体删除、分享过期时 public 接口返回不可访问状态。

### 13.2 第二阶段：匿名会话与 public 聊天接口

目标：在不登录的情况下，让分享页能够创建独立会话并发起问答。

主要任务：

1. 在会话模型中补充 `access_mode`、`agent_page_share_id`、`anonymous_visitor_id`、`visitor_token_hash` 等匿名分享会话字段。
2. 新增 public 创建匿名会话接口，返回 `session_id` 和一次性明文 `visitor_token`。
3. 新增 public 消息列表接口，使用 `session_id + X-Share-Session-Token` 校验会话归属。
4. 新增 public 分享问答接口，内部复用现有 Knowledge QA / Agent QA 服务，不新建第二套执行管线。
5. 新增 public continue stream / stop generation 适配，复用现有 SSE 事件格式。
6. 所有 public 问答请求都由后端根据 `share_code` 反解智能体配置，忽略访问者传入的设置字段。

交付标准：

1. 未登录浏览器打开分享页可以创建匿名会话并完成一轮问答。
2. SSE 返回格式与现有聊天页兼容，前端无需新增第二套流式解析逻辑。
3. 错误令牌不能读取或继续其他匿名会话。
4. 匿名请求无法切换模型、知识库、智能体或 Web Search。

### 13.3 第三阶段：前端分享页外壳与现有聊天页复用

目标：新增分享页入口，但最大程度复用现有聊天页面功能。

主要任务：

1. 新增 `/share/agents/:shareCode` 路由，配置 `requiresAuth: false`、`requiresInit: false`。
2. 新增 `AgentShareChatPage.vue` 作为薄外壳，只负责分享解析、匿名会话创建/恢复和运行时上下文组装。
3. 改造 `frontend/src/views/chat/index.vue`，增加 `runtimeContext` prop，在分享模式下切换到 public API。
4. 改造 `frontend/src/components/Input-field.vue`，增加 `runtimeContext` prop，保留原组件但锁定设置类入口。
5. 复用现有 `botmsg.vue`、`usermsg.vue`、`AgentStreamDisplay.vue`、`ChatDocumentArtifactCard.vue`、推荐问题、Markdown 渲染、自动滚动、停止生成和重试逻辑。
6. 在分享页外壳中不挂载 `frontend/src/views/platform/index.vue`，避免菜单、全局设置弹窗、命令面板进入分享页。

交付标准：

1. 分享页视觉与现有聊天体验一致，但没有平台菜单和设置入口。
2. 现有聊天页在 `/platform/chat/:chatid` 下行为不变。
3. 分享页可以复用现有消息渲染、流式输出、Markdown、artifact 卡片和附件展示。
4. 分享页输入区只显示提问和智能体允许的上传能力。

### 13.4 第四阶段：风控、审计与稳定性

目标：让匿名分享在可控成本和可审计边界内运行。

主要任务：

1. 增加分享级、IP 级、匿名会话级限流。
2. 增加匿名会话过期策略和 token 失效策略。
3. 增加访问日志和调用量统计，记录分享 ID、会话 ID、IP hash、UA hash、请求耗时、token 消耗。
4. 增加每日调用量、每日 token 量或失败率熔断配置。
5. 增加分享关闭后的即时失效处理，确保已打开页面的下一次请求失败。

交付标准：

1. 异常高频访问会被限流。
2. 分享关闭、过期、智能体删除后，匿名会话不能继续发问。
3. 管理员可以定位某个分享链接的调用量和异常情况。

### 13.5 第五阶段：测试与回归

目标：确保分享能力不破坏现有平台聊天，并覆盖匿名访问核心风险。

后端测试建议：

1. `CreateAgentPageShare_RequiresAgentOwnerOrManager`。
2. `PublicGetAgentPageShare_AllowsAnonymousAccess`。
3. `PublicChat_UsesSharedAgentConfigAndIgnoresOverrideFields`。
4. `PublicChat_RejectsInvalidShareSessionToken`。
5. `PublicChat_CannotAccessDisabledOrExpiredShare`。
6. `PublicChat_EnforcesAgentUploadCapabilities`。

前端测试建议：

1. 未登录访问 `/share/agents/:shareCode` 不跳转登录页。
2. 分享页挂载现有聊天组件并能正常显示流式消息。
3. 分享页不显示 AgentSelector、KnowledgeBaseSelector、模型选择器、Web Search 开关和设置入口。
4. 分享页刷新后能通过本地 `session_id + visitor_token` 恢复当前匿名会话。
5. 平台标准聊天页不受分享运行时上下文影响。

集成验收建议：

1. 创建智能体分享链接后，复制到无登录浏览器窗口可直接问答。
2. 修改智能体配置后，分享页下一轮问答按最新配置执行。
3. 关闭分享后，原链接和已创建匿名会话都不能继续使用。
4. 尝试在 public chat 请求中传入 `model_id`、`knowledge_base_ids`、`web_search_enabled` 等字段，后端不会采纳。

## 14. 待确认事项

以下事项建议在开发前由产品或需求方确认：

1. 匿名会话默认有效期，例如 24 小时、7 天或跟随浏览器本地缓存长期有效。
2. 分享页是否允许访问者上传附件，如果允许，文件类型是否仍完全跟随智能体配置。
3. 分享页是否需要展示“由谁分享”或“所属组织”信息。
4. 分享关闭后是否需要保留历史会话只读查看能力。
5. 分享链接是否需要支持手动设置过期时间、每日调用上限或密码保护。若启用密码保护，仍属于分享页访问凭证，不等同于平台登录认证。

在上述待确认项未新增约束的前提下，本方案默认采用“匿名免认证访问、只聊天不配置、行为完全跟随当前智能体配置、匿名会话令牌保护会话读写”的实现策略。
