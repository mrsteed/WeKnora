## 7. 关于剩余 git cherry “+” 提示的说明

截至本轮结束，`git cherry -v sync/dev-from-upstream backup/dev-20260428` 仍会显示若干 `+` 记录，但这些并不代表“还有未处理代码”。

原因是：

1. 对应主题已经在整合分支中落地。
2. 为保留当前分支逻辑和兼容 upstream 新接口，实际提交内容经过了冲突适配。
3. 因此生成的 patch 与原始 dev 提交不再完全等价，`git cherry` 会继续把原 commit 标记为 `+`。

换句话说，剩余 `+` 更多反映的是“patch 形态不同”，而不是“功能没有同步”。
