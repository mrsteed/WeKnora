# WeKnora 对外功能清单

## 1. 说明

本文档面向潜在用户、客户、合作伙伴和实施团队，用于说明 WeKnora 当前能够提供哪些可感知功能与平台能力。

本文档不再按照代码目录拆分，而是改为提供一份可直接对外使用的完整功能清单。在呈现上，会把用户可感知的产品功能、平台底层公共能力和多端交付能力统一放进同一张总表中，并通过“能力类型”和“模块归类”来组织。

编排思路参考了 [docs/20260618-01-项目功能清单.md](docs/20260618-01-项目功能清单.md) 中“建议作为对外介绍重点突出的能力组合”，重点围绕以下五条主线展开：

1. 知识导入链路。
2. Agent 能力链路。
3. 企业治理链路。
4. 多端交付链路。
5. 知识沉淀链路。

## 2. 项目对外定位

WeKnora 是一套面向企业知识场景的智能知识平台。它不仅支持把文档接入系统并进行智能问答，还能进一步提供 Agent 推理、外部工具编排、知识沉淀、权限治理、多端接入和工程化部署能力。

对外看，WeKnora 的价值可以概括为三句话：

1. 把分散的文档、FAQ 和外部知识源接入成统一的知识资产。
2. 让用户通过问答、Agent、搜索和 Wiki 多种方式消费知识。
3. 让平台具备企业级的权限、审计、部署和扩展能力。

## 4. 典型对外介绍主线

为了让外部用户更快理解系统能力，建议在介绍时重点围绕以下主线展开：

### 4.1 知识导入链路

从上传文档、导入 URL、接入外部知识源开始，经过文档解析、分块、索引构建，最终形成可问答、可搜索、可沉淀的知识资产。

### 4.2 Agent 能力链路

从创建 Agent、选择知识范围、挂载技能和工具，到通过 ReAct 推理循环执行多步骤任务，再到分享 Agent 页面给外部使用者，形成从配置到消费的完整链路。

### 4.3 企业治理链路

从登录认证、OIDC、租户与组织树管理，到角色权限控制和审计日志，形成一套适合企业环境落地的治理框架。

### 4.4 多端交付链路

同一套平台能力可以通过 Web 控制台、CLI、Go SDK、MCP Server、小程序和分享页多种形式对外提供，覆盖不同角色和使用场景。

### 4.5 知识沉淀链路

平台不仅回答问题，还支持把结果进一步沉淀为会话导出、长文档、Wiki 页面和知识图谱，让知识资产持续累积和复用。

## 5. 适合对外强调的能力结论

如果从用户价值角度总结，WeKnora 当前最适合对外强调的能力可以归纳为：

1. 能把多来源知识快速接入并组织成统一知识库。
2. 能通过 RAG、搜索和 Agent 提供可落地的知识消费能力。
3. 能把问答和文档结果进一步沉淀为 Wiki 和图谱型知识资产。
4. 能以企业级方式治理权限、成员、审计和部署环境。
5. 能通过 Web、CLI、SDK、MCP 和轻量端等多种方式对外交付能力。

从外部介绍视角看，WeKnora 不是单一聊天产品，而是一套“知识接入 + 智能消费 + 企业治理 + 多端交付 + 知识沉淀”一体化平台。
